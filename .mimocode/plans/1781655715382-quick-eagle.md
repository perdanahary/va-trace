- 4 orders duplicated under `New`, 2 under `In Production`, 3 under `Ready to Ship`
- `Waiting` status has zero coverage
- Seed data relies on localStorage fallback — user wants clean seed-only data

- OR-2026-715187, OR-2026-901234, OR-2026-570006, OR-2026-445566, OR-2026-300001, OR-2026-570008

### 2. Add `Waiting` status order

Create OR-2026-570010 with items having `status: "Waiting"` — represents orders waiting on materials, client approval, or vendor assignment.

### 3. Modify `src/lib/orderStore.ts`

Change `readStoredOrders()` to **always return the seed data** — remove localStorage persistence logic. The `useOrders()` hook will return `mockOrders` directly without localStorage reads/writes.

Remove: `readStoredOrders()`, `writeStoredOrders()`, localStorage event listeners, `cachedOrders`/`cachedStorageValue` variables.

Replace with: `useOrders()` returns `mockOrders` directly via `useRef` (no external store needed).

Keep `upsertOrder`, `appendOrders`, `saveOrders` etc. as in-memory-only operations that update a module-level array (for runtime CRUD during the session, but no persistence).
