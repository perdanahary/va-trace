# Plan: Partial Production Batch Creation & Unfinished Items Display

## Context
During production phase, the platform blocks batch creation entirely — even when some production jobs are complete (ready qty > 0). The user needs to ship partially finished items while remaining items continue production. Additionally, the production jobs table should visually separate finished vs in-progress jobs.

## Changes (all in `src/pages/vendor/VendorOrderDetail.tsx`)

### 1. Show "Create Shipment Batch" during production phase

**Location:** `renderWorkflowActions` (lines 327-355)

Currently the `isProductionPhase` branch only renders "Update Production" + dropdown. Add the batch creation button when ready items exist.

**Replace lines 327-355** with:

### 2. Add "Ready for Shipping" stat during production phase

**Location:** Overview tab stats grid (around line 458)

After the existing summary stats, conditionally render an extra stat card when `isProductionPhase && totalReadyQty > 0`:

```tsx
<div className="grid gap-3 sm:grid-cols-2 xl:grid-cols-4">
  {viewModel.summaryStats.map((stat) => (
    <OverviewStat key={stat.label} label={stat.label} value={stat.value} hint={stat.hint} color={stat.color} />
  ))}
  {isProductionPhase && totalReadyQty > 0 ? (
    <OverviewStat
      label="Ready for Shipping"
      value={`${totalReadyQty} pcs`}
      hint={`${Math.round((totalReadyQty / viewModel.order.quantitySummary.orderedQuantity) * 100)}% of ordered`}
      color="text-success"
    />
  ) : null}
</div>
```
